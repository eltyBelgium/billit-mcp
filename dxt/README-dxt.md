# Billit DXT (Desktop Extension)

This directory contains the manifest and assets for the one-click `.dxt`
installer for **Claude Desktop**.

## How this DXT works

It's a **thin launcher**, not a bundle. The manifest's run command is:

```
uvx --from git+https://github.com/eltyBelgium/billit-mcp@v0.1.0 billit-mcp-server --transport stdio
```

When Claude Desktop starts the server, `uvx` fetches this repo at the pinned
tag and resolves dependencies **for the user's own platform and Python** — so
the same `.dxt` works on macOS, Windows, and Linux. The bundle itself contains
only `manifest.json` + `icon.png` (~2 KB).

### Why a launcher instead of bundling deps

`pydantic-core` ships as a compiled binary (`_pydantic_core.<abi>-<os>.so`). A
bundled DXT would be locked to the single OS **and** Python minor version it was
built against — broken for almost everyone. The launcher sidesteps that by
letting `uv` pick the correct wheels at install time.

## Prerequisite: `uv`

The user must have [`uv`](https://docs.astral.sh/uv/) installed (it provides
`uvx`). One-liners:

```bash
# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh
# Windows (PowerShell)
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

This is the same tool the rest of the install docs lead with, so most users
who already run MCP servers will have it.

> Once `billit-mcp-server` is published to PyPI, the `--from git+...` can be
> dropped to `uvx billit-mcp-server` — slightly faster cold start. The git form
> works today without any PyPI release.

## Build

```bash
# From the repo root — Node 20+ for the DXT CLI
npx -y @anthropic-ai/dxt pack ./dxt billit-mcp.dxt
```

`.dxtignore` keeps `make_icon.py`, this README, and any stray `server/` dir out
of the bundle. CI (`.github/workflows/publish-dxt.yml`) runs this on every
GitHub Release and attaches `billit-mcp.dxt` as an asset.

## Updating the pinned tag

The manifest pins `@v0.1.0`. When you cut a new release, bump that ref in
`manifest.json` (and the `version` field) so the DXT installs the new code.

## Icon

`icon.png` is generated by `make_icon.py` (pure stdlib, no Pillow). Replace it
with real branding anytime — keep it 256×256.
